Opérations MySQL

Présentation

Le plugin Opérations MySQL permet d'effectuer diverses tâches de maintenance des tables DotClear à l'aide d'une interface Web.

Actuellement, vous disposez de la possibilité :

  • d'optimiser vos tables,
  • de sauvegarder vos tables dans un fichier de commandes SQL,
  • de restaurer vos tables depuis un fichier de commandes SQL produit par la fonctionnalité de sauvegarde

Optimisation

Il faut voir une base de données MySQL comme un ensemble de fichiers. En multipliant les suppressions ou modifications de leur contenu, les tables se fragmentent.

L'opération Optimisation va donc exécuter une requête OPTIMIZE TABLE sur l'ensemble des tables utilisées par DotClear afin de compacter celles-ci.

Dans la plupart des installations, vous n'avez pas à utiliser Optimisation. Même si vous faites beaucoup de mises à jour sur vos billets, vous pouvez vous contenter de lancer cette opération mensuellement.

Cette opération n'a aucun impact sur l'intégrité de vos données. Vous pouvez donc l'utiliser sans risques.

Sauvegarde

On ne vous rappellera jamais assez combien il est important de faire des sauvegardes régulières.
Avec l'opération Sauvegarde, vous n'aurez plus de raisons valables de pas avoir ce réflexe.

Cette opération crée un "dump" complet de vos tables DotClear. Nous appelons "dump" un fichier de commandes SQL contenant les instructions nécessaires pour reconstituer votre base de données DotClear.
Ce fichier est crée sous forme gzippée afin de minimiser sa taille.

Vous pouvez également utiliser ce fichier depuis l'application phpMyAdmin si vous le souhaitez.

Télécharger directement
Cette option indique si vous souhaitez que le plugin enregistre le dump sur le serveur ou si vous préférez récupérer directement ce fichier.
Par défaut, cette case est cochée et le fichier généré vous est directement transmis, sans enregistrement sur le serveur.
En décochant cette option, le fichier sera enregistré dans un sous-répertoire mysql/ du dossier share/ de votre DotClear.

Restauration

Il s'agit de l'opération inverse de la sauvegarde. Cette fonction vous permet de recharger un fichier de sauvegarde en cas de problème avec votre base MySQL.

Ce devrait être l'opération que vous utiliserez le moins, du moins, faut-il l'espérer.

La restauration exécute l'ensemble des commandes SQL figurant dans le fichier de dump. Ce qui signifie que toutes les données subsistant encore dans les tables DotClear à ce moment là seront écrasées.
Ceci est également valable pour les comptes utilisateurs.

Fichier de dump à restaurer
Utilisez le bouton parcourir... pour sélectionner, sur votre machine personnelle, le fichier dump à restaurer.
